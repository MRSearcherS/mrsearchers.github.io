# DECISIONS.md

# Принятые решения и правила проекта

Дата обновления: 2026-07-11 23:37:09

## Назначение файла

Этот файл хранит важные решения, чтобы не возвращаться к одним и тем же вопросам.

---

## Решение 1. Основной адрес сайта

Основной адрес сайта:

https://mrs.spb.ru/

Именно его использовать в публичных материалах, ссылках и QR-кодах.

---

## Решение 2. GitHub Pages

Сайт размещается через GitHub Pages.

GitHub Pages используется как статический хостинг.

---

## Решение 3. Домен

Кастомный домен:

mrs.spb.ru

Файл CNAME должен содержать:

    mrs.spb.ru

---

## Решение 4. QR-код

QR-код должен вести на:

https://mrs.spb.ru/

Для печати предпочтительно использовать SVG.

Не использовать временный github.io-адрес для финального QR-кода.

---

## Решение 5. Структура сайта

Использовать структуру с исходниками:

- src/template.html
- src/build.js
- src/locales/*.json

Готовые HTML-файлы генерируются сборкой.

---

## Решение 6. Сборка

После изменения шаблона или локалей запускать:

    node src/build.js

---

## Решение 7. Языковые версии

Поддерживать несколько языков:

- русский;
- английский;
- китайский.

---

## Решение 8. Команды для пользователя

Поскольку пользователь работает в Windows, команды по умолчанию давать для PowerShell.

---

## Решение 9. Безопасность правок

Не использовать опасные массовые замены и удаления без предварительной проверки.

Особенно осторожно работать с:

- HTML-блоками;
- регулярными выражениями;
- командами, которые перезаписывают файлы;
- Git-командами, меняющими историю.

---

## Решение 10. Если контекст неясен

Ассистент не должен додумывать.

Если не хватает данных, нужно попросить:

- актуальный файл;
- вывод команды;
- скриншот;
- конкретный путь;
- git status.

---

## Решение 11. Текущий код важнее памяти

Если есть расхождение между тем, что ассистент помнит, и тем, что лежит в текущих файлах проекта, верить текущим файлам.

---

## Решение 12. Историю переписки не хранить полностью

Не нужно хранить всю переписку целиком.

Нужно хранить:

- текущий контекст;
- актуальный код;
- важные решения;
- открытые задачи;
- краткий журнал работ.

Старый код, ошибочные варианты и устаревшие команды не переносить в постоянные файлы, если они больше не используются.

---

## Решение 13. Не использовать агрессивные regex-удаления

Перед удалением HTML-блока:

1. Найти блок.
2. Показать найденный фрагмент.
3. Уточнить границы удаления.
4. Предложить безопасный способ.
5. Только потом удалять.

---

## Решение 14. Проверять реальные имена файлов

Не предполагать пути к картинкам или другим файлам.

Перед исправлением путей сначала проверить реальные имена файлов.
